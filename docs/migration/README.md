# 迁移指南 (MigrationGuides)

本目录存放所有版本间的迁移指南。

当框架发生重大变更（通常由一个RFC驱动）导致向后不兼容时，必须在此处提供一份详细的迁移指南。

一份好的迁移指南应包括：
-   **变更概述**: 简要说明发生了什么变化以及为什么需要这次变更。
-   **受影响的范围**: 明确指出哪些API、配置或使用模式受到了影响。
-   **手动迁移步骤**: 提供清晰的、分步的指导，告诉用户如何手动更新他们的代码以适配新版本。
-   **自动化迁移工具**: （如果提供）说明如何使用Codemod等自动化脚本来完成迁移。
