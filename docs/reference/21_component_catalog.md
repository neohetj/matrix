# 学习Matrix组件目录 (ComponentCatalog)

本文档是Matrix框架中所有可用标准节点的官方目录。该文件由Agent自动扫描源码生成。

> **给开发者的提示:**
> 你可以使用“源码锚点”列中的唯一ID，在IDE中进行全局搜索，以快速定位到该组件的具体实现代码。

## 1. Action (动作类) (ActionNodes)
动作类节点用于执行核心的业务逻辑和流程控制。

| 源码锚点 (Source Anchor) | 节点名称 (Name) | 描述 (Description) |
| :--- | :--- | :--- |
| `action/exprSwitch` | Expression Switch | 根据可配置的表达式路由消息。 |
| `action/flow` | Flow | 同步执行一个子规则链，并在其完成后继续流程。 |
| `action/forEach` | For Each | 遍历列表（来自`msg.DataT`或表达式），并为每一项执行子规则链。 |
| `functions` | Function | 根据名称执行一个预先注册的函数。 |
| `action/log` | Log | 使用指定的日志级别将消息记录到控制台。 |

## 2. Endpoint (入口类) (EndpointNodes)
入口类节点是规则链的起点，负责从外部世界接收事件或请求。

| 源码锚点 (Source Anchor) | 节点名称 (Name) | 描述 (Description) |
| :--- | :--- | :--- |
| `endpoint/http` | HTTP Endpoint V2 | 接收HTTP请求，并基于统一的定义触发规则链。 |
| `endpoint/inotify` | Inotify Endpoint | 主动监听文件系统事件（如写入、创建）。 |
| `endpoint/redisStream` | Redis Stream Endpoint | 主动消费来自Redis Stream的消息。 |

## 3. External (外部资源类) (ExternalNodes)
外部资源类节点通常是共享节点，用于管理与外部系统的连接。

| 源码锚点 (Source Anchor) | 节点名称 (Name) | 描述 (Description) |
| :--- | :--- | :--- |
| `external/dbClient` | DB Client | 提供一个共享的数据库连接池 (`*sqlx.DB`)。 |
| `external/neo4jClient` | Neo4j Client | 提供一个用于连接Neo4j数据库的共享客户端。 |
| `external/redisClient` | Redis Client | 提供一个共享的Redis客户端 (go-redis)。 |

## 4. Functions (可执行函数) (FunctionNodes)
这些是可被 `functions` 节点调用的预注册业务函数。

| 源码锚点 (Source Anchor) | 函数名称 (Name) | 描述 (Description) |
| :--- | :--- | :--- |
| `redisCommand` | Redis Command | 执行一个或多个Redis命令。 |
| `sqlQuery` | SQL Query | 使用共享的DB客户端执行一条SQL查询。 |
| `startTransaction` | Start Transaction | 启动一个数据库事务并将其存储在上下文中。 |
| `commitTransaction` | Commit Transaction | 提交一个存储在上下文中的事务。 |
| `rollbackTransaction` | Rollback Transaction | 回滚一个存储在上下文中的事务。 |

## 5. Link (流程连接类) (LinkNodes)
连接类节点用于在可视化编排时提供流程上的便利。

| 源码锚点 (Source Anchor) | 节点名称 (Name) | 描述 (Description) |
| :--- | :--- | :--- |
| `flow/endpointLink` | Endpoint Link | 一个虚拟节点，用于在UI中链接到另一个入口点。 |

<!-- qa_section_start -->
> **问：如何使用`functions`节点来调用一个函数？**
> **答：** 在你的规则链DSL中，创建一个类型为`functions`的节点。在其`configuration`中，设置`functionName`字段为上表中你想调用的函数ID（例如`"redisCommand"`），然后根据该函数所需的业务参数填写`business`对象。
<!-- qa_section_end -->
